Gestión de Riesgos en Proyectos

Resumen de conceptos clave y metodologías para la identificación, análisis y control de riesgos

Definición de Riesgo

Un riesgo, en el contexto de un proyecto, es un evento o condición incierta que, si se produce, tiene un efecto positivo o negativo en uno o más de los objetivos del proyecto.

Riesgos Negativos (Amenazas)

Son aquellos que podrían impactar de manera negativa algún objetivo del proyecto. La acción principal es minimizarlos.

Riesgos Positivos (Oportunidades)

Son aquellos que podrían impactar de manera positiva algún objetivo del proyecto. La acción principal es maximizarlos.

La causa del riesgo es la situación que lo introduce, es decir, lo que produce el riesgo o cuál es su fuente.

Proceso de Administración de Riesgos

La gestión de riesgos sigue un proceso estructurado de siete pasos fundamentales:

  1. Establecer el contexto

    Identificar el contexto en el cual se encuentra el proyecto, estableciendo criterios de evaluación y la estructura de análisis.

  2. Identificar Riesgos

    Conocer todos aquellos riesgos que puedan impactar de manera positiva o negativa al proyecto y/o a la organización.

  3. Analizar Riesgos

    Generar un análisis de cada riesgo para conocer el nivel de afectación que tendría en el proyecto.

  4. Evaluar Riesgos

    Comparar el riesgo estimado contra criterios predeterminados, estableciendo un ranking de priorización.

  5. Tratar Riesgos

    Seleccionar e implementar opciones apropiadas para manejar el riesgo (evitar, reducir, compartir o aceptar).

  6. Comunicar y Consultar

    Comunicar, informar y consultar a los interesados internos y externos en cada etapa del proceso.

  7. Monitorear y Revisar

    Definir y utilizar mecanismos para monitorear y revisar el desempeño del proceso de gestión de riesgos.

Estructuras de Desglose

Estructura de Desglose de Trabajo (EDT/WBS)

La Estructura de Desglose de Trabajo consiste en la descomposición del producto resultante del proyecto en paquetes de trabajo individuales. Esta descomposición se realiza de manera jerárquica.

  • Es un documento del proceso de planificación
  • Es creado por el equipo del proyecto
  • Permite subdividir el trabajo en porciones más pequeñas
  • Ayuda a conocer los entregables en cada etapa
  • Facilita la identificación detallada de riesgos

Estructura de Desglose de Riesgos (RBS)

Es una estructura similar al WBS, pero su función es agrupar u organizar distintas fuentes de riesgos, generando una categorización de ellas.

Para cada proyecto se debe crear su propio RBS, debido a que no todos tienen las mismas características o recursos.
  • Identifica y clasifica las fuentes de riesgos
  • Establece un orden en la identificación
  • Facilita la organización por categorías
  • Permite una visión estructurada de los riesgos

Carta Gantt

Un cronograma es el listado de tareas ordenadas de manera cronológica, necesarias para el logro de un objetivo o realización de un proyecto. La Carta Gantt es una representación gráfica de estas actividades que permite conocer lo que se deberá hacer y saber el estado actual de las actividades para hacer seguimiento de ellas.

Cómo realizar una Carta Gantt

  1. Identificar todas las tareas necesarias para completar el proyecto
  2. Establecer la duración de cada tarea
  3. Determinar las dependencias entre tareas (qué debe completarse antes de iniciar otra)
  4. Asignar recursos a cada tarea
  5. Establecer fechas de inicio y fin para cada tarea
  6. Representar gráficamente las tareas en una línea de tiempo
  7. Actualizar el diagrama a medida que avanza el proyecto
Diagrama de Gantt
  • Diagrama de barras
  • Muy conveniente para proyectos en curso
  • Muestra la duración de las tareas
Diagrama PERT
  • Diagrama de red
  • Muy conveniente para la etapa de planificación
  • Proporciona fechas límites para cada actividad
Herramientas para crear diagramas de Gantt: Microsoft Project, GanttProject (gratis) y otras aplicaciones en línea.

Análisis de Riesgos

Después de identificar los riesgos, es importante realizar un análisis para conocer cómo será su impacto en el proyecto. Este análisis se puede realizar de dos maneras:

Análisis Cualitativo

Implica analizar los riesgos evaluando la probabilidad de que ocurran y el impacto de ellos.

  • Es subjetivo
  • Prioriza los riesgos de manera individual
  • Es rápido de desarrollar
  • Se puede utilizar en todo tipo de proyectos
  • Se evalúan en base a su probabilidad e impacto
Análisis Cuantitativo

Se basa en modelos matemáticos, midiendo la manera en que un riesgo puede afectar los objetivos de un proyecto.

Métodos principales:
  • Simulación de Monte Carlo: Genera valores de entrada al azar para simular posibles resultados
  • Estimación PERT: Utiliza tres valores (optimista, más probable y pesimista) para estimar duración o costo
  • Árbol de decisión: Representa situaciones con varios escenarios para elegir la mejor alternativa

Estrategias de Respuesta a Riesgos

Una vez identificados y analizados los riesgos, se deben establecer estrategias para responder a ellos:

Para Riesgos Negativos (Amenazas)
  • Evitar:

    Busca que el riesgo no suceda. Puede requerir inversión de recursos.

  • Transferir:

    Transfiere la responsabilidad de la gestión a otro. No elimina el riesgo, solo lo transfiere.

  • Mitigar:

    Busca bajar la probabilidad de que un riesgo ocurra y/o bajar su impacto.

  • Aceptar:

    Se utiliza cuando no hay estrategia válida o el riesgo es muy bajo. Puede requerir reserva de contingencia.

Para Riesgos Positivos (Oportunidades)
  • Explotar:

    Explotar las opciones para asegurar que se concrete la oportunidad.

  • Mejorar:

    Busca mejorar las posibilidades de que ocurra una oportunidad.

  • Compartir:

    Se usa cuando hay una oportunidad pero no se tiene la capacidad o experiencia para realizarlo solo.

  • Aceptar:

    Se acepta el impacto positivo que generará el riesgo si se hace presente.

Monitoreo de Riesgos

Durante las diferentes fases de un proyecto, se monitorean los riesgos para ver si se producen o no. De activarse uno de ellos, se debe poner en acción los mecanismos de respuesta planificados.

Métodos de Monitoreo
  • Reuniones: Involucran al equipo y ayudan a actuar tempranamente
  • Revaluación de riesgos: Permite una actualización oportuna
  • Auditoría de riesgos: Evalúa si las respuestas han sido efectivas
Más Métodos
  • Análisis de tendencias: Muestra la evolución del proyecto
  • Medición del desempeño: Permite saber cómo se está desempeñando el proyecto
  • Análisis del uso de reservas: Sirve para conocer cuánta reserva queda
Si no se realiza el monitoreo de los riesgos de manera efectiva, todos los pasos anteriores no servirán. Es importante formalizar el proceso con reportes periódicos.
